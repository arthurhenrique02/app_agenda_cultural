## Codebase Patterns
- Use `StrEnum` (from `enum` stdlib) instead of `(str, enum.Enum)` â€” ruff UP042 enforces this.
- Alembic migrations cannot be autogenerated without a live DB. Create them manually when PostgreSQL is unavailable; offline SQL generation still works for testing.
- Migration files need blank line between stdlib imports and third-party imports (ruff I001); run `ruff check --fix` after writing migrations.

- Backend uses `uv` as Python package manager (not pip/poetry). Run `uv run pytest`, `uv run ruff check .`, `uv run black .`.
- pytest-asyncio is configured with `asyncio_mode = "auto"` in pyproject.toml - no need for explicit `@pytest.mark.asyncio` decorator (though it still works).
- Use `ASGITransport` from httpx to test FastAPI endpoints without starting a real server.
- Dev dependencies go in `[dependency-groups] dev = [...]` section (uv's format), not `[project.optional-dependencies]`.
- Use `aiosqlite` (dev dep) with `sqlite+aiosqlite:///:memory:` for in-memory async DB tests â€” avoids needing a real Postgres instance in unit/api tests.
- SQLAlchemy engine masks passwords in `str(engine.url)`; use `engine.url.render_as_string(hide_password=False)` to compare with the raw settings URL.
- Override `db_module.AsyncSessionLocal` in tests to swap the session factory without touching the app-level `get_db` dependency.
- Alembic env.py uses `asyncio.run(run_async_migrations())` + `async_engine_from_config` for online mode; offline mode is sync and works for SQL generation tests.
- Test Alembic offline SQL generation by running subprocess `alembic upgrade head --sql` with a dummy DATABASE_URL (no real DB needed).
- All model classes must inherit from `app.models.base.Base` (DeclarativeBase); import `app.models` in env.py for autogenerate to detect all tables.
- Shared test fixtures live in `backend/tests/conftest.py`. Use `db_session` for an isolated AsyncSession (SQLite in-memory) and `async_client` for an httpx client wired to the FastAPI app with DB overridden.
- The `async_client` fixture internally overrides `db_module.AsyncSessionLocal` to redirect all DB operations to the test SQLite engine.
